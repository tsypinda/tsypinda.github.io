## Git — это больно, но надо

Если ты только начинаешь работать с Git, скорее всего, первые ощущения — смесь недоумения и лёгкой паники. Команды кажутся нелогичными, ошибки пугают, а сообщения типа "fatal: не понимаю, что ты хочешь" выбивают из колеи.

Я сам через это прохожу. Но есть три команды, без которых никуда. Освоив их, ты уже сможешь нормально работать над проектами и сохранять код на GitHub.

### Что такое Git одной фразой

Git — это система, которая следит за изменениями в твоих файлах. Как «Контроль версий» в Google Docs, только для кода и через терминал.

### Команда №1: git add

Перед тем как сохранить изменения, их нужно **добавить в очередь** (в терминах Git — в «индекс» или «staging area»).

```bash
# Добавить конкретный файл
git add README.md

# Добавить все изменения в текущей папке
git add .

# Добавить все файлы с определённым расширением
git add *.js
